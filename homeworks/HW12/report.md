Основная сложность прогнозирования заключается в улавливании динамически меняющейся амплитуды суточных сезонных колебаний и аномальных пиков, для чего могут потребоваться адаптивные модели или глубокое обучение, способное учитывать долгосрочные зависимости

## 4. Temporal split и признаки

### 4.1. Разбиение по времени

- Как именно выполнен `train / validation / test`: разбиение выполнено хронологически, данные разделены строго по порядку времени без перемешивания
- Какие доли или интервалы использовались: 70% / 15% / 15%
- Почему `random split` здесь некорректен: утечка данных из будущего, искажение оценки качества, нарушение целостности временных зависимостей

### 4.2. Признаки для baseline-моделей

- Какие лаги использовались: использовались лаги с периодами [1, 2, 3, 5, 7, 10, 14, 21, 28, 35, 42], то есть значения целевой переменной со сдвигом на 1, 2, 3, 5, 7, 10, 14, 21, 28, 35 и 42 шага назад
- Какие rolling-признаки использовались: для окон [3, 7, 14, 28] были построены скользящее среднее (roll_mean), стандартное отклонение (roll_std), максимум (roll_max), минимум (roll_min) и размах (roll_range), причем все признаки рассчитывались с предварительным сдвигом на 1 шаг для предотвращения утечки данных
- Какие календарные признаки использовались: из даты были извлечены час (hour), день недели (day_of_week), день месяца (day_of_month), месяц (month), бинарный признак выходного дня (is_weekend), а также циклические тригонометрические кодировки часа (hour_sin, hour_cos) и дня недели (dow_sin, dow_cos)
- Как обрабатывались пропуски после построения лагов/rolling: после создания всех признаков строки с пропущенными значениями (NaN), возникшими из-за сдвигов и оконных операций, были удалены с помощью метода dropna() перед обучением моделей
- Использовалось ли масштабирование и где именно оно обучалось: масштабирование применялось в двух местах: для признаков модели Ridge масштабирование (StandardScaler) обучалось на обучающей выборке и применялось к валидационной и тестовой; для нейросетевой модели GRU масштабирование целевой переменной также обучалось на обучающем наборе и использовалось для трансформации валидационных и тестовых данных перед формированием окон

## 5. Модели и эксперименты (B1, B2, B3, R1)

Опишите коротко и сопоставимо:

- B1 (`naive-last`): в качестве прогноза используется последнее наблюденное значение (предыдущий час), что соответствует простейшему базовому методу "наивного" прогнозирования без использования каких-либо признаков или обучения
- B2 (`moving-average`): прогноз вычисляется как скользящее среднее за последние window наблюдений, при этом оптимальный размер окна (window=3) был подобран по минимальной MAE на валидационной выборке среди вариантов 3, 7, 14, 28, 42
- B3 (`ridge-lag-features`): используется линейная регрессия с L2-регуляризацией (Ridge) на наборе из 45 признаков, включающих лаги (1–42), rolling-статистики, экспоненциальные скользящие средние, разности и календарные признаки; модель обучается на масштабированных данных
- R1 (`gru-forecast`): применяется рекуррентная нейронная сеть с архитектурой GRU (2 слоя, hidden_size=64, dropout=0.2) для прогнозирования по окну из 28 предыдущих нормализованных значений целевой переменной

Дополнительно:

- Основная метрика выбора лучшей модели: MAE (средняя абсолютная ошибка) на валидационной выборке
- Дополнительные метрики: RMSE (корень из среднеквадратичной ошибки) и MAPE (средняя абсолютная процентная ошибка) также вычислялись и сохранялись для всех моделей
- `window_size` для `GRU`: 28
- Batch size: 128
- Максимальное число эпох: 40
- Optimizer: Adam
- Learning rate: 1e-3
- Как сохранялась лучшая модель: d gроцессе обучения на каждой эпохе проверялось значение val_loss. Если оно оказывалось минимальным за все время обучения, веса модели сохранялись в переменную best_state. По завершении обучения модель загружала сохраненные веса, после чего итоговая лучшая модель была сохранена в файл artifacts/best_gru.pt
