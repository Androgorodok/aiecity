# HW08-09 – PyTorch MLP: регуляризация и оптимизация обучения

## 1. Кратко: что сделано

- Был выбран датасет EMNIST, так как MNIST слишком простой, а также его не получается подключить
- В части A мы боролись с переобучением через регуляризацию (Dropout, BatchNorm), а в части B — настраивали гиперпараметры оптимизатора (LR, SGD) для улучшения сходимости модели.

## 2. Среда и воспроизводимость

- Python: 3.13.0
- torch / torchvision: 2.10.0 / 0.25.0
- Устройство (CPU/GPU): CPU
- Seed: 42
- Как запустить: открыть `HW08-09.ipynb` и выполнить Run All.

## 3. Данные

- Датасет: EMNIST Balanced
- Разделение: train/val/test — 80/20 из train=True части с фиксированным SEED=42 и отдельный test из train=False
- Трансформации (transform): ToTensor() + Normalize((0.5,), (0.5,)) для масштабирования в диапазон [-1, 1]
- Комментарий (1-3 предложения): изображения 28×28 в градациях серого, инвертированные цвета (тёмный символ на светлом фоне). Задача сложнее классического MNIST из-за схожести символов ('l'/'1', 'S'/'5') и большего числа классовю Это требует регуляризации и тщательного подбора гиперпараметров.

## 4. Базовая модель и обучение

- Модель MLP (кратко): 3 скрытых слоя ([512, 256, 128]), активация ReLU, Dropout отключен (так как выбран базовый вариант для E4)
- Loss: CrossEntropyLoss
- Базовый Optimizer (для части A): Adam (lr=1e-3, weight_decay=0.0)
- Batch size: 64
- Epochs (макс): 10
- EarlyStopping: patience=5, метрика мониторинга — val_accuracy

## 5. Часть A (S08): регуляризация (E1-E4)

- E1 (base): 3 скрытых слоя [512, 256, 128], без Dropout/BatchNorm
- E2 (Dropout): как E1 + Dropout(p=0.3)
- E3 (BatchNorm): как E1 + BatchNorm (не применялось, модель идентична E1)
- E4 (EarlyStopping): лучший из (E2/E3) → выбрана база (E1) + EarlyStopping(patience=5)

## 6. Часть B (S09): LR, оптимизаторы, weight decay (O1-O3)

- O1: LR слишком большой (Adam, lr=0.5)
- O2: LR слишком маленький (Adam, lr=1e-5)
- O3: SGD+momentum (momentum=0.9) + weight_decay=1e-4 (lr=2e-3)
