# Итоговый проект по курсу «Инженерия Искусственного Интеллекта»

В этой папке находится итоговый мини-проект по курсу.  
Проект должен демонстрировать применение методов и инструментов инженерии ИИ: работу с данными, модели, пайплайны, сервис, эксперименты и (по возможности) воспроизводимость.

---

## 1. Паспорт проекта

- **Название проекта:** `Классификация заболеваний листьев кукурузы по изображениям`
- **Автор:** `Войтковский Андрей Иванович`
- **Группа:** `ИКБО-50-24`
- **Контакт:** `androgorodok@gmail.com`

- **Краткое описание:**  
  `Проект представляет собой систему классификации болезней кукурузы по изображениям листьев на основе глубокого обучения. Используется архитектура ResNet18, предварительно обученная на ImageNet, и датасет Corn or Maize Leaf Disease Dataset, содержащий 4 класса: Blight, Common_Rust, Gray_Leaf_Spot и Healthy. Модель достигает точности 94,3% на тестовой выборке и развёрнута как веб-сервис на FastAPI с REST API для загрузки изображений и получения предсказаний с вероятностями по всем классам. Проект включает полный пайплайн: загрузку и предобработку данных, обучение, сохранение артефактов и инференс через API.`

## 3. Требования и установка

### 3.1. Требования

- Python `>= 3.10` (желательно 3.10).
- Установленные системные зависимости (если есть: например, `git`, `make` и т.п.).

### 3.2. Установка окружения

Сценарий:

```bash
# Перейти в папку проекта
cd project

# Создать виртуальное окружение
python -m venv .venv

# Активировать окружение:
# Windows:
.venv\Scripts\activate
# Linux / macOS:
source .venv/bin/activate

# Установить зависимости
pip install --upgrade pip
pip install -r requirements.txt
```

### 3.3. Скачивание датасета

Перед обучением необходимо скачать данные:

```bash
# Перейти в папку проекта
cd project

# Запустить скрипт скачивания датасета
python scripts/download_dataset.py
```

## 4. Как запустить проект

### 4.1. Запуск обучения модели

```bash
cd project # если не находитесь в этой папке
source .venv/bin/activate
python -m src.train
```

### 4.2. Запуск сервиса (API/веб-интерфейс)

```bash
cd project
source .venv/bin/activate      # при необходимости
python -m src.service          # пример: FastAPI/Flask сервис
```

После запуска сервис доступен по адресу, если его не поменяли в `configs/servive_config.yaml`:

`http://localhost:8000`

Swagger UI, если его не поменяли в `configs/servive_config.yaml`:

`http://localhost:8000/docs`

Проверить работу можно в Swagger UI (эндпоинт `/predict`) или через CURL (путь к файлу можно поменять):

```bash
curl -X POST ^
  "http://localhost:8000/predict" ^
  -F "file=@data/data/Blight/Corn_Blight (18).jpg"
  ```
